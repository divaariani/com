Description

This application is a privacy-first, secure mobile communication platform built with Flutter. It balances robust hardware-linked security with a frictionless user experience. The app features secure device-bound registration, biometric authentication with automatic fail-safes, dynamic time-bound peer discovery (QR/Hash), and a rich-media communication suite (Chat, Voice, and Video calls). A key differentiator is its dual-messaging architecture, offering standard direct messaging (Legacy Mode) alongside completely anonymized, auto-named group chats (Ghost Mode). The entire real-time messaging infrastructure is powered by a custom-engineered, team-optimized Flutter Chat SDK.

Problem

When designing modern communication applications, developers and users typically face three core challenges:

  1. Identity & Access: Standard username/password combinations are easily compromised, and accounts are often accessed from unauthorized, malicious devices.
  2. Privacy in Group Dynamics: Users frequently want to discuss topics in groups without exposing their personal identities, phone numbers, or names to other members.
  3. Complex SDK Implementations: Off the shelf communication SDKs are often bloated, rigid, and difficult to maintain, leading to messy codebases and slow development cycles.
  4. Friction vs Security: High security features (like long passwords and manual keys) often ruin the onboarding experience, causing users to abandon the app.

Solution

To solve these challenges, the platform implements specific, high performance architectural choices:

  1. Device Bound Security & Self Recovery
    The app binds the user's account to their physical smartphone hardware using a seamless unique Device ID during registration. If a user forgets their password, they rely on a self 5 words random recovery phrase, eliminating the need to store sensitive data on centralized recovery servers.
  2. Frictionless UX
    Onboarding utilizes biometrics (Fingerprint/Face ID) for instant access. To prevent hardware related lockouts, the system features an intelligent fallback mechanism that redirects users to a secure password login after 3 failed biometric attempts.
  3. Secure Peer Discovery
    Instead of static, leak prone invite links, contacts are added via proximity QR scanning or unique Hash Codes. These codes utilize a Time based One Time Password (TOTP) logic that expires every 3 minutes, ensuring secure and intentional connections.
  4. Identity Anonymization (Ghost Mode)
    To protect user privacy in group settings, the app features Ghost Mode. The system automatically generates the group name and completely anonymizes all participants, allowing open communication without identity exposure.
  5. Custom Engineered Developer Experience
    Instead of wrestling with rigid third party code, the development team redesigned and customized a Flutter Chat SDK. This encapsulation streamlined the codebase, simplified state management, and made adding features (like pinning, starring, and replying to messages) incredibly efficient.

Methodology

Our engineering methodology focuses on code modularity, strict environment separation, and rigorous security standards:

  1. Modularization (Internal Packages): Architected into 6 isolated internal packages to ensure reusability and clean separation of concerns (including network client, chat SDK, local auth bridge, notifications, and call wrappers).
  2. Environment Strategy: Utilizes 3 separate Build Flavors to strictly isolate Development, Staging, and Production environments.
  3. Security & Privacy: Hardened security using libsodium for E2EE, biometric authentication, screenshot prevention, and dynamic group anonymization (Ghost Mode).
  4. Reliability & Testing: Robust offline first mechanism with automatic data synchronization upon reconnection, backed by 28 targeted test files (focused on cryptography) and centralized logging via Talker.

Architecture Code

The technical architecture is built for high performance, predictable state management, and high scale maintainability:

  1. Architectural Pattern: Strict Clean Architecture (Domain, Data, Presentation layers) combined with the BLoC Pattern. Robust codebase scale including 27 entities, 6 repositories, and 52 use cases.
  2. State Management & Routing: Implements flutter_bloc + equatable for reactive state handling, alongside go_router managing a complex navigation network of 80+ routes.
  3. Data & Real time Stream: Integrates Socket.IO for instant messaging, LiveKit for voice/video calls, and Drift ORM (SQLite) for structured local data.
  4. Design Patterns: Leverages Repository, Use Case, Strategy Pattern (for multi-provider E2EE architecture), Provider, and Observer Pattern for centralized logging.
