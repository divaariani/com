  • Description: PondMate is a mobile application integrated with IoT technology, specifically MQTT, serving as a controller for a specialized device designed to manage ponds. This application enables users to monitor and control the connectivity status and on/off functions of the device via their mobile devices. Additionally, it includes a scheduling feature for setting desired times for device connection and disconnection.

Problem

Managing ponds or water features has been a manual and time-consuming task. Without automated systems, pond owners face challenges in maintaining optimal conditions, such as monitoring water levels, controlling pumps, and ensuring timely filtration. Additionally, the lack of remote access makes it difficult to address issues promptly, leading to potential risks for aquatic life and increased maintenance costs.

Solution

This app offers a comprehensive solution by integrating IoT technology with a mobile application. The system comprises a specialized device that utilizes MQTT for communication and control. Through the mobile app, users can remotely monitor and manage their pond equipment, including checking connectivity status and turning devices on/off. Moreover, the scheduling feature allows users to set specific times for device connection and disconnection, enhancing convenience and efficiency. With MQTT integration in both the Flutter app and the backend, PondMate provides a user-friendly and efficient solution for pond management.

Methodology

The Waterfall methodology applied in the development of the application to ensure smooth analysis, design, implementation, testing, and distribution processes, resulting in an application that enables employees to easily attendance record, enhancing operational flexibility, and efficiency.

Architecture Code

The project adopts a MVC architecture within the Flutter Dart programming framework, featuring a well-organized folder structure. It includes dedicated folders such as Controllers, responsible for handling API calls and control; Pages, housing and viewing UI components; Models, responsible for handling datas; Utils, serving as a centralized hub for managing global variables and session management; Themes, responsible for handling design components.
